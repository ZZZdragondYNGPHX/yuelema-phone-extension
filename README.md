# 约了吗小手机 v0.1.23（阶段 45 AI 运行进度与操作弹窗生命周期修复）

这是现代现实都市「约了吗」MVU 角色卡的配套 SillyTavern UI 扩展。软件层承担推荐、线上短文本私聊、匹配、群组浏览、角色创作与面基约定；现实见面、约会及复杂长文本剧情仍由酒馆正文推进。

当前成果已通过本地静态检查与 Node 回归，尚未完成 SillyTavern 真机验收，不得据此宣称已在酒馆内发布可用。

> 连接预设、提示词预设和功能绑定可保存在本浏览器；首次使用会写入 14 份可编辑的默认提示词预设，重开连接页会自动载入默认连接预设。API Key 按连接预设 ID 保存到当前浏览器的独立缓存，绝不导出、写入 MVU 或回显；可在连接页删除当前 Key。真实认证、CORS、模型接口与扩展生命周期仍待目标酒馆版本验证。

## 当前已实现

- 悬浮圆形入口、可关闭且支持鼠标/触摸拖动的小手机窗口，以及首页 / 匹配 / 消息 / 群组 / 我的五导航。
- SFW / NSFW 五击解锁门：连续点击“关于软件”五次只显示右侧点击式开关，实际切换由开关通过受控 MVU 管线完成；内容尺度不替代明确同意、成年人校验或边界记录。
- 首页只保留毛玻璃候选角色卡：公开资料、关键词与喜欢 / 不喜欢 / 收藏 / 刷新四个操作；快速模型候选的昵称必须是自然人的个人姓名，不能是玩家、AI、系统、职业或概念标签。喜欢 / 收藏 / 不喜欢成功保存后自动生成下一位，刷新只生成下一位且不改变权重，取消收藏不自动刷新。首页的喜欢/不喜欢只学习公开标签偏好，不会建立匹配或会话；收藏把对象留入收藏夹。空池卡片的刷新调用 `recommendation_refresh` 绑定的快速模型。
- 角色创建入口已迁移到“我的”。“我的 → 设置 → 图片管理”提供浏览器本地图片库：可导入本地 PNG/JPEG/WebP（压缩后保存）或 HTTP/HTTPS 图片链接，显示预览与加载失败状态；每张图可维护 `-5…5` 整数关键词权重，通过桌面右键或移动端长按打开“编辑匹配关键词”，并可在编辑窗口删除图片。图片管理页右上角新增“设置”按钮，打开“图片匹配设置”并绑定 `image_match` 的连接 / 提示词预设。`image_match` 只读取候选人的公开资料与图片库已有关键词；LLM 失败时回退本地关键词裁决。首页候选卡背景、首页头像和候选公开资料头像使用最高匹配图片，无结果时保留原渐变与姓名首字头像。
- 公开标签长期偏好、本地双层匹配、匹配会话与短文本私聊；模型只提出受限关系变化，程序校验并提交。
- 首页空池生成、首页刷新、灵魂匹配和语音匹配在候选公开资料通过严格校验后，会在同一次用户操作中非阻塞启动图片匹配 LLM；图片选择不等待、不阻塞对应模块 LLM 的角色生成、受控 MVU 写回或会话创建，拒绝和超时只影响展示回退。
- “消息”一级页是独立会话列表；点入后进入深色霓虹风格的私聊详情，含联系人卡、聊天气泡、会话内回复中状态与加长的对话区域。
- “消息”一级页支持按昵称或最近一条可见消息搜索；私聊详情右上角提供 `…` 更多菜单。“清空聊天记录”承接旧会话删除语义，仅移除会话并按关系状态安全处理已匹配/已拉黑关系；“删除角色”会在一次受控 Patch 中删除该角色在推荐、角色池、会话、面基记录和群组中的全部 MVU 引用，但不会删除浏览器图片库、角色模板等设备侧数据，也不会回退 UID 计数器。AI 的 `replies` 会拆成多条消息气泡；达到已读不回或拉黑阈值时分别写入固定系统提示，拉黑后会话只读。输入栏右侧改为 Telegram 风格纸飞机：左键发送，右键打开工具栏；工具栏当前只有“约定面基”，点击后才展开原面基表单。点击私聊对象头像会打开其公开资料。发送与面基仍分别复用既有受控桥，面基只填入正文输入框、不自动发送。
- 个性化内容推荐设置位于“设置 → 隐私权限设置 → 个性化内容推荐管理 → 个性化内容偏好”；关键词权重保存在当前设备的开放学习库。首页刷新会把全部关键词及 `-5…5` 权重、玩家公开资料与性别 / 性取向双向匹配条件交给既有推荐模型；AI 可自由生成新鲜、安全的公开标签，不受固定主题表限制。候选成功走完受控 MVU 写回后，未知公开标签自动以 `0` 收录，已有词保留当前权重；首页喜欢 / 收藏 / 不喜欢再按公开标签同步 `+3 / +1 / -3`（范围 `-5…5`）。关闭个性化后不会继续累积。收藏夹仅有“取消收藏”和“发起私聊”：后者由本地算法合成公开心动名片兼容度（性别/性取向、城市、意图、年龄段、距离）与关键词权重兼容度，再与角色创建页互动节奏的拒绝阈值比较；通过才建立私聊，未通过只保留婉拒结果、不创建会话。灵魂匹配和语音匹配分别使用设备关键词权重与“临时权重覆盖同名本地权重”规则，随后由本地公开资料 / 关键词评分裁决；只有分数达到角色的取消匹配阈值才物化为已匹配关系、建立私聊并写入“已互相喜欢”，未达到时只记录婉拒 / `已取消` 角色，不进入消息，也不复用收藏角色。语音匹配原始输入与临时权重不会持久化或显示。
- 群组以小程序中心提供聊天群与论坛：均只读取明确成年对象的公开投影，并分别调用独立绑定生成内存草稿；无发布按钮、无 MVU 写入。“进入已有私聊”也只做本地导航。
- 面基在已匹配前提下生成并保存约定记录，再仅填入 `#send_textarea`；扩展不会自动点击或发送，玩家发送后由正文推进现实剧情。
- “我的”提供恋爱氛围的角色创建界面与模板库：手动创建、导入导出、占位 / URL / 本地压缩头像、AI 完善补全与 AI 完整创作。个人资料编辑页不再保存头像引用；“我的”页头像可点击打开本地菜单，支持导入、填写图片链接或移除，且独立保存在当前浏览器，不写入公开资料、MVU、导出或提示词。页面右上角和两张 AI 卡片都提供独立“设置”入口；两种 AI 调用分别绑定，不再共用一个角色创作预设。AI 只返回内存草稿，玩家审核并点击“验证并登记”后才进入受控 Patch。
- OpenAI-compatible 非机密连接预设、提示词预设（`depth / order / position / enabled / content`）、模型拉取及按功能绑定。首次加载会把推荐人物、私聊、AI 补全、AI 完整创作各自的 SFW / NSFW 默认预设写入本地提示词预设列表，之后可直接查看、编辑、删除或导出；连接预设与提示词预设字段相互隔离；API Key 单独保存在当前浏览器缓存，调用端会按预设 ID 自动恢复。
- 首页、匹配、消息、群组、AI 补全与完整创作均可通过右上角“设置”分别绑定连接／提示词预设。绑定按 SFW / NSFW 分开保存；切换内容模式后，已打开的设置弹窗和所有入口都会显示当前模式对应的一套绑定，提示词选择器不会混入另一模式。旧角色创作绑定会兼容迁移到两个新入口。
- 提示词预设使用 Prompt Manager 风格的条目树：根节点是预设，下面按 `before_character_definition` / `after_character_definition` 分支展示条目；叶节点显示启用状态、depth、order 和正文摘要，可直接编辑、启停、上下移动。每个预设都有“是否为 NSFW”标记；首次加载提供推荐、私聊、角色补全、完整创作、灵魂匹配、语音匹配各一套 SFW / NSFW 默认预设。导入导出格式保持兼容。
- 连接预设支持 `json` 普通响应、`stream` SSE 真流式和 `pseudo_stream` 假流式。拉取模型列表使用独立的宽松探测校验，Model 可留空；正式保存和对话请求仍要求已选模型。SSE 会处理跨 chunk 分行、`[DONE]`、文本数组、超时、取消和 2 MiB 响应上限；假流式只改变完整响应后的展示元数据，不能绕过服务端输出上限。首页推荐刷新会至少申请 2048 Token，并兼容单层 JSON 代码围栏；推荐、灵魂匹配、语音关键词 / 候选以及角色 AI 完善 / 完整创作都会在用户提示词预设之后附加不可删除的系统级 JSON 结构合同。SFW / NSFW 都要求完整成年候选结构，仅公开标签规则按模式区分。
- 收藏主动私聊、灵魂匹配、语音匹配与通用 AI 调用接入恋爱软件视觉状态：进行中为两颗心之间的电波连接，成功为双心依偎/心跳，婉拒与失败分别显示独立的疏离/心碎动画。操作提示均提供右上角 `×` 和底部关闭按钮；关闭、切页、收起小手机或销毁后，迟到结果不会重新弹窗或强制跳转；被关闭的匹配/私聊提示会以“已关闭，结果未展示”终结其控制台记录。
- “我的 → 设置 → 控制台”显示本次小手机会话的安全运行进度，最多保留 30 条纯内存记录；覆盖首页首位生成、刷新、反馈后的下一位、群聊草稿、论坛帖子草稿及既有匹配/角色创作调用。仅展示固定公开名称、状态、提示与时间，不持久化，也不显示 Key、URL、UID、Patch、`stat_data`、Prompt、原始响应、堆栈、隐藏资料、关系分或阈值。
- 消息列表、私聊联系人与消息气泡、收藏夹、匹配列表和群聊成员均已接入统一头像视图；支持安全 HTTP/HTTPS、图片 Data URL 与姓名首字回退。远程头像的网络隐私和长会话 Data URL 性能仍需真机验证。

## 状态、隐私与写入边界

- 所有可注册、匹配、私聊和面基对象必须明确成年；NSFW 不自动等于同意。
- 普通推荐、匹配、群组只读公开资料；私聊仅在已匹配时读取允许的仅好友资料；隐藏资料、实际年龄、内部阈值、关系分、私密草稿和 API Key 不得进入普通 DOM、导出预览或模型错误信息。
- UI 不能直接写 MVU 状态。唯一提交链为：

```text
readLatestState → build…Patch → validateControlledPatchAgainstState
→ Mvu.parseMessage → Mvu.replaceMvuData → VARIABLE_UPDATE_ENDED
```

- 空卡刷新、普通刷新、角色登记、匹配、私聊和面基记录均使用受控 RFC 6902 JSONPatch；禁止任意 UI/模型提供 JSON Pointer、UID 或 Patch。

## MVU 运行前置与安全降级

- 本扩展的完整读写能力要求当前聊天已加载兼容的 MagVarUpdate/MVU 提供方，且 `window.Mvu` 至少暴露 `getMvuData`、`parseMessage`、`replaceMvuData` 及对应的 `Mvu.events.VARIABLE_UPDATE_ENDED`。
- `JS-Slash-Runner` / `TavernHelper.waitGlobalInitialized(''Mvu'')` 只提供全局就绪通知，**不是** MVU 实现本身；它不能代替上述 provider。
- 扩展激活时若 MVU 尚未出现，会先保持只读的“MVU 状态暂不可读”界面；若已存在的 provider 随后发布可读 `Mvu`，扩展只会重新绑定监听并刷新投影，**不会**因等待动作写入任何聊天状态。
- 不可用或不完整时，扩展不会退回 `TavernHelper.insertOrAssignVariables`、通用变量 API 或直接改聊天元数据。所有状态变更仍必须走本文定义的 MVU 受控链。
- 角色卡中的远程脚本导入属于第三方网络代码。仅在使用者明确同意并确认其来源、版本与网络策略后启用；本扩展自身不会主动加载、替换或下载 MVU provider。
